## 1. 为什么要使用数据库

### 问题：内存存储的缺点

```python
# 内存存储
questions_db: list[dict] = []
```

**缺点**：
- 服务器重启后数据丢失
- 无法存储大量数据（内存有限）
- 无法多用户同时访问

### 解决方案：数据库

```python
# 数据库存储
db.add(question)  # 保存到数据库文件
db.commit()       # 确认保存
```

**优点**：
- 数据永久保存（存在硬盘文件里）
- 可以存储大量数据
- 支持多用户同时访问

## 2. 数据库是什么

### 类比

**数据库** = 永久存储数据的地方（硬盘上的文件）

**类比**：
- 内存 = 白板（写上去，擦掉就没了）
- 数据库 = 笔记本（写上去，一直在）

### 数据库类型

| 类型 | 说明 | 例子 |
|------|------|------|
| SQLite | 轻量级，数据存一个文件 | 本项目使用 |
| MySQL | 中型数据库，需要安装服务 | 网站常用 |
| PostgreSQL | 大型数据库，功能强大 | 企业常用 |

### 本项目使用 SQLite

**原因**：
- 不需要安装额外软件
- 数据存一个文件 `ai_interview.db`
- 适合开发和学习

## 3. SQL 是什么

### 定义

**SQL** = Structured Query Language（结构化查询语言）

**作用**：用来操作数据库的语言

### 类比

**SQL** = 数据库的"语法"

就像 Python 是写程序的语法，SQL 是操作数据库的语法。

### 基本 SQL 语句

#### 创建表

```sql
CREATE TABLE questions (
    id INTEGER PRIMARY KEY,
    title TEXT,
    category TEXT,
    difficulty INTEGER
);
```

**解释**：
- `CREATE TABLE`：创建一张表
- `questions`：表名
- `id INTEGER PRIMARY KEY`：id 是整数，是主键（唯一标识）
- `title TEXT`：title 是文本类型
- `difficulty INTEGER`：difficulty 是整数类型

#### 插入数据

```sql
INSERT INTO questions (title, category, difficulty) 
VALUES ('什么是闭包？', 'Python', 3);
```

**解释**：
- `INSERT INTO`：插入数据
- `questions`：表名
- `(title, category, difficulty)`：要插入的列
- `VALUES (...)`：要插入的值

#### 查询数据

```sql
-- 查询所有数据
SELECT * FROM questions;

-- 查询指定条件
SELECT * FROM questions WHERE category = 'Python';

-- 查询单条数据
SELECT * FROM questions WHERE id = 1;
```

**解释**：
- `SELECT`：查询数据
- `*`：所有列
- `FROM questions`：从 questions 表
- `WHERE`：条件

#### 更新数据

```sql
UPDATE questions 
SET title = '新标题', difficulty = 5 
WHERE id = 1;
```

**解释**：
- `UPDATE`：更新数据
- `questions`：表名
- `SET`：设置新值
- `WHERE`：指定更新哪条数据

#### 删除数据

```sql
DELETE FROM questions WHERE id = 1;
```

**解释**：
- `DELETE FROM`：删除数据
- `questions`：表名
- `WHERE`：指定删除哪条数据

## 4. ORM 是什么

### 定义

**ORM** = Object-Relational Mapping（对象关系映射）

**作用**：让你用 Python 代码操作数据库，不用写 SQL

### 类比

**ORM** = 翻译官

把 Python 代码翻译成 SQL 语句。

### 对比

#### 不用 ORM（写 SQL）

```python
# 需要写 SQL 语句
cursor.execute("INSERT INTO questions (title) VALUES ('什么是闭包？')")
cursor.execute("SELECT * FROM questions WHERE id = 1")
```

#### 用 ORM（写 Python）

```python
# 用 Python 代码
question = Question(title="什么是闭包？")
db.add(question)
db.commit()

# 查询
question = db.query(Question).filter(Question.id == 1).first()
```

### 为什么用 ORM？

| 优点 | 说明 |
|------|------|
| 不用写 SQL | 用 Python 代码就行 |
| 代码更清晰 | 对象比 SQL 字符串更易读 |
| 防止 SQL 注入 | ORM 自动处理安全问题 |
| 跨数据库 | 换数据库不用改代码 |

## 5. SQLAlchemy 基础

### 什么是 SQLAlchemy

**SQLAlchemy** 是 Python 最流行的 ORM 框架。

**作用**：让你用 Python 代码操作数据库。

### 安装

```bash
pip install sqlalchemy aiosqlite
```

### 核心概念

#### 1. 引擎（Engine）

```python
from sqlalchemy import create_engine

# 创建数据库引擎
engine = create_engine("sqlite:///./ai_interview.db")
```

**作用**：连接数据库的"插头"

#### 2. 会话（Session）

```python
from sqlalchemy.orm import sessionmaker

# 创建会话工厂
SessionLocal = sessionmaker(bind=engine)

# 创建会话
db = SessionLocal()
```

**作用**：和数据库交互的"窗口"

#### 3. 模型（Model）

```python
from sqlalchemy.orm import DeclarativeBase
from sqlalchemy import Column, Integer, String

# 创建基类
class Base(DeclarativeBase):
    pass

# 定义模型
class Question(Base):
    __tablename__ = "questions"
    
    id = Column(Integer, primary_key=True)
    title = Column(String(200))
    category = Column(String(50))
    difficulty = Column(Integer)
```

**作用**：定义数据库表的结构

#### 4. 创建表

```python
# 创建所有表
Base.metadata.create_all(bind=engine)
```

**作用**：根据模型创建数据库表

## 6. 常用操作

### 创建数据

```python
# 创建对象
question = Question(
    title="什么是闭包？",
    category="Python",
    difficulty=3
)

# 添加到数据库
db.add(question)

# 提交（确认保存）
db.commit()

# 刷新对象（获取数据库生成的 id）
db.refresh(question)

print(question.id)  # 1
```

### 查询数据

```python
# 查询所有
questions = db.query(Question).all()

# 查询单条（根据 id）
question = db.query(Question).filter(Question.id == 1).first()

# 条件查询
python_questions = db.query(Question).filter(
    Question.category == "Python"
).all()
```

### 更新数据

```python
# 查询要更新的数据
question = db.query(Question).filter(Question.id == 1).first()

# 修改数据
question.title = "新标题"
question.difficulty = 5

# 提交
db.commit()
```

### 删除数据

```python
# 查询要删除的数据
question = db.query(Question).filter(Question.id == 1).first()

# 删除
db.delete(question)

# 提交
db.commit()
```

## 7. 常见问题

### Q1：数据存在哪里？

**A**：存在 SQLite 数据库文件里。

**文件位置**：`week2_fastapi/ai_interview.db`

### Q2：为什么重启后数据还在？

**A**：因为数据存在硬盘文件里，不是内存里。

**类比**：
- 内存 = 白板（擦掉就没了）
- 数据库 = 笔记本（一直在）

### Q3：`db.commit()` 是什么？

**A**：确认保存数据到数据库。

**类比**：就像你写完作业，按"保存"按钮。

### Q4：`db.refresh()` 是什么？

**A**：从数据库重新读取数据。

**用途**：获取数据库自动生成的 id。

### Q5：`db.query(Question).filter().first()` 是什么？

**A**：查询符合条件的第一条数据。

**分解**：
- `db.query(Question)`：查询 Question 表
- `.filter(Question.id == 1)`：条件是 id = 1
- `.first()`：取第一条

### Q6：什么是主键（Primary Key）？

**A**：唯一标识每条数据的字段。

**特点**：
- 每条数据的主键都不一样
- 通常是 `id` 字段
- 数据库自动生成

### Q7：什么是外键（Foreign Key）？

**A**：引用其他表的字段。

**例子**：

```python
class Answer(Base):
    __tablename__ = "answers"
    
    id = Column(Integer, primary_key=True)
    question_id = Column(Integer, ForeignKey("questions.id"))  # 外键
    content = Column(Text)
```

**作用**：关联两张表

## 8. 项目中的文件说明

### database.py

**作用**：数据库连接配置

**核心代码**：

```python
# 数据库文件路径
SQLALCHEMY_DATABASE_URL = "sqlite:///./ai_interview.db"

# 创建引擎
engine = create_engine(SQLALCHEMY_DATABASE_URL)

# 创建会话工厂
SessionLocal = sessionmaker(bind=engine)

# 创建基类
class Base(DeclarativeBase):
    pass

# 获取会话的函数
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

### models.py

**作用**：定义数据库表结构

**核心代码**：

```python
class Question(Base):
    __tablename__ = "questions"
    
    id = Column(Integer, primary_key=True)
    title = Column(String(200))
    category = Column(String(50))
    difficulty = Column(Integer)
```

### main.py

**作用**：API 接口，使用数据库

**核心代码**：

```python
# 创建数据
@app.post("/questions")
def create_question(question: QuestionCreate, db: Session = Depends(get_db)):
    db_question = Question(title=question.title, ...)
    db.add(db_question)
    db.commit()
    db.refresh(db_question)
    return db_question

# 查询数据
@app.get("/questions")
def list_questions(db: Session = Depends(get_db)):
    questions = db.query(Question).all()
    return questions
```
